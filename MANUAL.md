## 1. 必要なもの

| 種類 | バージョン | 用途 |
|------|----------|------|
| **Node.js** | 18 以上（推奨 LTS） | フロントエンド実行 |
| **Python** | 3.10〜3.12 推奨 | 譜面の自動生成 |
| **WAVファイル** | `.wav` のみ | プレイしたい曲 |
| **モダンブラウザ** | Chrome / Edge / Firefox / Safari 最新版 | ゲーム画面 |

> Python 3.13 でも動く可能性はありますが、Windows 環境では librosa の一部機能で問題が出ることがあります。3.13を使う場合は [トラブルシューティング](#10-トラブルシューティング) を参照してください。

## 2. 初回セットアップ

### 2-1. フロントエンドの依存関係をインストール

ターミナルで:

```bash
cd e:/neon-beat
npm install
```

`node_modules/` が作られればOK。

### 2-2. Python依存ライブラリをインストール

```bash
pip install -r tools/requirements.txt
```

`librosa` `numpy` `soundfile` がインストールされます。数分かかることがあります。

### 2-3. （推奨）Python 3.12 の仮想環境を使う場合

3.13 をメインで使っていて、このプロジェクトだけ 3.12 を使いたい場合:

```bash
# Python 3.12 を別途インストールしてから
cd e:/neon-beat
py -3.12 -m venv .venv

# Git Bash の場合
source .venv/Scripts/activate

# PowerShell の場合
.venv\Scripts\Activate.ps1

# 仮想環境内で依存をインストール
pip install -r tools/requirements.txt
```

次回以降、ターミナルを開き直したら毎回仮想環境をアクティベートしてください。

## 3. 曲を追加する

### 3-1. WAVファイルを準備

**ファイル名のルール:**

- 拡張子は **`.wav`** のみ
- ファイル名がそのまま曲名になります
- 使える文字: 英数字、`_`、`-`、スペース
- `_` はゲーム画面でスペースとして表示されます

**例:**

| ファイル名 | 表示される曲名 |
|----------|--------------|
| `My_Cool_Song.wav` | `My Cool Song` |
| `Track 01.wav` | `Track 01` |
| `Neon_Disco_Night.wav` | `Neon Disco Night` |

mp3 などの場合は事前に WAV へ変換してください:

```bash
# ffmpeg がある場合
ffmpeg -i input.mp3 output.wav
```

または **Audacity**（無料）で読み込んで「書き出し → WAV」でも可能です。

### 3-2. import/ フォルダにコピー

`e:/neon-beat/import/` フォルダに WAV ファイルを置きます（エクスプローラからドラッグ&ドロップでOK）。複数曲をまとめて置いても大丈夫です。

### 3-3. 譜面を自動生成

```bash
cd e:/neon-beat
python tools/batch_import.py
```

実行例:

```
[batch_import] found 2 file(s) in import/
[generate_chart] analyzing: import/My_Cool_Song.wav
[generate_chart] OK: My_Cool_Song  BPM=128.0  duration=195.3s  E=120/N=250/H=450/X=700
[generate_chart] analyzing: import/Track_01.wav
[generate_chart] OK: Track_01  BPM=140.0  duration=210.5s  E=130/N=270/H=480/X=750
[batch_import] done. processed=2 skipped=0 failed=0 song-list total=2
```

`E/N/H/X` がそれぞれ EASY/NORMAL/HARD/EXPERT のノーツ数です。
1曲あたり数秒〜30秒程度かかります（曲長に比例）。

### 3-4. 生成結果

`public/songs/` に以下が作られます:

```
public/songs/
├── song-list.json              ← 曲一覧マニフェスト
├── My_Cool_Song/
│   ├── audio.wav               ← コピーされた音源
│   └── chart.json              ← 自動生成された譜面
└── Track_01/
    ├── audio.wav
    └── chart.json
```

### 3-5. オプション

```bash
# 既存曲も再解析（譜面を作り直す）
python tools/batch_import.py --force

# 別フォルダからインポート
python tools/batch_import.py --import-dir ./other_folder

# 単一ファイルだけ処理
python tools/generate_chart.py path/to/song.wav
```

## 4. ゲームを起動する

```bash
cd e:/neon-beat
npm run dev
```

ターミナルに以下が表示されます:

```
➜  Local:   http://localhost:5173/
➜  Network: http://192.168.x.x:5173/
```

ブラウザで **http://localhost:5173/** を開いてください。

> **モバイルでプレイする場合**: 同じWi-Fi内のスマホで `Network` のURL（`http://192.168.x.x:5173/`）を開けばタッチ操作でプレイできます。

ゲームを止めるには、ターミナルで **Ctrl+C** を押してください。

## 6. 画面の説明

### 6-1. タイトル画面

- 「NEON BEAT」のロゴ
- **START** ボタン → 曲選択画面へ
- **SETTINGS** ボタン → 設定パネルを開く
- 背景にはコンボ10相当のアイドル動作でイコライザーがゆらゆら踊っています

### 6-2. 曲選択画面

- インポート済みの曲がカード形式で並びます
- 各カードに曲名・アーティスト・BPM・長さ・各難易度のノーツ数が表示
- 曲が0件の場合は案内メッセージが出ます

### 6-3. 難易度選択画面

| 難易度 | レベル目安 | 特徴 |
|--------|----------|------|
| **EASY** | 3 | レーン0と3のみ、ビートの1/2 |
| **NORMAL** | 5 | 全ビート、4レーン使用 |
| **HARD** | 8 | ビート + オンセット、同時押しあり |
| **EXPERT** | 10 | 16分音符グリッド + オンセット、高密度 |

### 6-3-2. ゲーム開始演出（4秒の白フェード + 5秒のカウントダウン）

難易度を決定すると、以下の流れでゲームが始まります:

| 時刻 | 演出 |
|------|------|
| 0.0s | 「ピカーン」SE再生 + 画面全体が白く染まり始める |
| 0.0–2.0s | 白オーバーレイが透明 → 不透明（曲選択画面が見えなくなる） |
| 2.0s | 裏で GameScreen がマウント・チャートとオーディオを読み込む |
| 2.0–4.0s | 白オーバーレイが不透明 → 透明（ゲーム画面がフェードイン） |
| 4.0–4.5s | タイトルと難易度バッジが登場 |
| 6.0–9.0s | 巨大カウントダウン **3 → 2 → 1** |
| 9.0s | BGM 再生 & プレイ開始 |

### 6-4. ゲーム画面

```
┌──────────────────────────────────────────────┐
│ [SCORE]   [COMBO]   [LEVEL] [♪ 曲名] [✕] │  ← HUD
│ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ │  ← プログレスバー
│                                              │
│     ┌───────────────────────────┐            │
│     │   ↓   ↓   ↓   ↓         │            │
│     │   ■   ■   ■   ■  ノーツ  │            │
│     │ ──────────────────────── │  ← 判定ライン
│     │  [D] [F] [J] [K]        │            │
│     └───────────────────────────┘            │
└──────────────────────────────────────────────┘
```

### 6-5. 判定

| 判定 | 許容誤差 | スコア計算 |
|------|---------|-----------|
| **PERFECT** | ±50ms | 100 × (1 + コンボ × 0.1) |
| **GREAT** | ±100ms | 75 × (1 + コンボ × 0.1) |
| **GOOD** | ±150ms | 50 × (1 + コンボ × 0.1) |
| **MISS** | それ以外 | 0（コンボリセット） |

コンボが続くほどスコア倍率が上がります。

### 6-6. ディスコレベル

コンボに応じて背景演出が派手になります:

| レベル | コンボ | 演出 |
|--------|--------|------|
| 0 | 0–4 | 控えめ |
| 1 | 5–14 | ディスコライト出現 |
| 2 | 15–24 | レーザー出現 |
| 3 | 25–39 | グリッドがゴールドに |
| 4 | 40+ | 全開・最大輝度 |

### 6-7. 曲終了演出（4.5秒の余韻 + ランク表示）

曲が自然に終わったあと、以下の流れでリザルト画面に遷移します:

| 時刻（曲終了基準） | 演出 |
|------------------|------|
| 0.0s | 曲が終わる |
| 0.0–0.5s | ランクラベルが画面中央に大スラムイン（ALL PERFECT / AMAZING / GREAT / GOOD など） |
| 0.5–1.0s | サブタイトル（"BRILLIANT" / "NICE PLAY" など）がスライドアップ |
| 1.0–1.5s | 精度パーセントが表示 |
| 1.5–4.5s | ランクラベルが脈動しながら静かに表示（余韻を楽しむ時間） |
| 4.5s | リザルト画面へ遷移 |

### 6-8. ランク評価

精度（accuracy）に基づくランク:

| ランク | 条件 | サブタイトル | 色 |
|--------|------|------------|-----|
| **ALL PERFECT** | 全ノーツPERFECT | FLAWLESS! | イエロー |
| **FULL COMBO** | ミス0 | NO MISS! | イエロー |
| **AMAZING** | 95%以上 | BRILLIANT | イエロー |
| **EXCELLENT** | 90%以上 | WELL DONE | シアン |
| **GREAT** | 80%以上 | NICE PLAY | シアン |
| **GOOD** | 70%以上 | KEEP IT UP | パープル |
| **NICE** | 60%以上 | NOT BAD | パープル |
| **OK** | 50%以上 | TRY HARDER | ピンク |
| **TRY AGAIN** | 50%未満 | PRACTICE! | ピンク |

精度の計算式: `(PERFECT×100 + GREAT×75 + GOOD×50) / (totalNotes×100) × 100%`

### 6-9. リザルト画面

- 曲名 + 難易度
- 総合スコア（中央に大きく）
- MAX COMBO
- 判定内訳（PERFECT / GREAT / GOOD / MISS）
- 精度
- AUTO PLAY だった場合は「AUTO PLAY」バッジ表示
- **RETRY / SELECT / TITLE** ボタン

## 7. 設定項目

タイトル画面または曲選択画面の **SETTINGS** ボタンから開けます。
変更は即座に反映され、`localStorage` に自動保存されます（キー: `neonbeat.settings.v2`）。

| 項目 | デフォルト | 範囲 | 説明 |
|------|----------|------|------|
| **NOTE SPEED** | 400 px/s | 200〜800 | ノーツの落下速度 |
| **JUDGE OFFSET** | 0 ms | -100〜+100 | 音ズレ補正（マイナスで早めに判定） |
| **BGM VOLUME** | **100%** | 0〜100% | 音楽の音量 |
| **SE VOLUME** | **35%** | 0〜100% | 効果音の音量（フォールバックの「ピコ」音は控えめに調整済み） |
| **AUTO PLAY** | OFF | ON / OFF | デバッグ用・自動プレイ |

### JUDGE OFFSET の調整方法

「ノーツより少し早く / 遅く押している気がする」と感じたら:

- **早く感じる場合**（押すタイミングが先行）: マイナス方向に調整
- **遅く感じる場合**（押すタイミングが遅れる）: プラス方向に調整

5〜10ms ずつ動かして自分に合う値を探してください。

## 8. 効果音をカスタマイズする

デフォルトでは判定時の効果音は **OscillatorNode で動的生成** されます（ピコピコ音）。
お気に入りのSEを使いたい場合、以下のファイル名で `public/se/` フォルダに置いてください:

| ファイル名 | タイミング |
|----------|----------|
| `hit_perfect.wav` | PERFECT判定時 |
| `hit_great.wav` | GREAT判定時 |
| `hit_good.wav` | GOOD判定時 |
| `miss.wav` | MISS判定時 |
| `combo_milestone.wav` | コンボ25/50/100到達時 |

ファイルが存在すれば自動で読み込まれ、無ければフォールバック音が使われます。

## 10. トラブルシューティング

### 「曲がありません」と表示される

`import/` フォルダに WAV を置いた後、`python tools/batch_import.py` を実行してください。
`public/songs/song-list.json` が存在し、`songs` 配列が空でないか確認してください。

### 音が出ない

ブラウザの自動再生ポリシーにより、ユーザー操作後でないと音が再生できません。
タイトル画面の **START** ボタンをクリックしてから曲を選んでください。

### `python: command not found`

Python がパスに通っていません。`py --version` を試すか、Python を再インストールして「Add Python to PATH」にチェックを入れてください。

### `pip install` でエラー

```bash
python -m pip install --upgrade pip
pip install -r tools/requirements.txt
```

### librosa のインストールに失敗（特に Python 3.13）

Python 3.13 で問題が出る場合の対処:

**(1) numba を最新化**
```bash
pip install --upgrade numba
pip install -r tools/requirements.txt
```

**(2) 削除された標準モジュールを補う**
```bash
pip install standard-aifc standard-sunau audioop-lts
pip install -r tools/requirements.txt
```

**(3) それでもダメなら Python 3.12 を別途インストール**

公式 https://www.python.org/downloads/ から 3.12 系をインストールし、仮想環境を作成:

```bash
py -3.12 -m venv .venv
source .venv/Scripts/activate    # Git Bash
# または
.venv\Scripts\Activate.ps1       # PowerShell

pip install -r tools/requirements.txt
```

### `Microsoft Visual C++ Build Tools が必要` と言われる

Python 3.12 を使うほうが速いです（3.12 用のプリビルド wheel が揃っているため）。

### ノーツが流れない / カクつく

- ブラウザのハードウェアアクセラレーションを有効に
- 別タブを閉じる
- DevTools (F12) のコンソールにエラーが出ていないか確認

### 音とノーツがズレる

設定画面の **JUDGE OFFSET** で調整してください（[7. 設定項目](#7-設定項目) 参照）。

### ポート 5173 が使われている

```bash
npx vite --port 5174
```

など別ポートで起動できます。

## 11. 仕組みの概要

### 譜面生成（Python / librosa）

1. WAVを読み込み
2. **ビート検出** (`librosa.beat.beat_track`) でビート位置を取得
3. **オンセット検出** (`librosa.onset.onset_detect`) で音の立ち上がりを取得
4. **STFT スペクトラム解析** で周波数帯域ごとのエネルギーを計算
5. 難易度ごとにノーツを配置

#### レーン割り当て（HARD / EXPERT）

各ノーツの時刻におけるスペクトラムを4つの帯域に分割し、**1ビンあたりの平均エネルギー** が一番大きい帯域のレーンを選択:

| レーン | 色 | 帯域 | 鳴りやすい音 | 重み |
|--------|-----|------|------------|------|
| 0 | ピンク | 20–200 Hz | キック・ベース | ×1.10 |
| 1 | シアン | 200–800 Hz | スネア・低域ボーカル | ×1.0 |
| 2 | イエロー | 800–2500 Hz | ボーカル・リード | ×1.0 |
| 3 | パープル | 2500–6000 Hz | ハイハット・シンバル | ×0.80 |

帯域は対数スケールで分割し、合計エネルギーではなく **平均** で比較しているため、帯域幅が広い高域に偏らないようになっています。さらにレーン3を×0.80・レーン0を×1.10と重み付けして、現代曲の明るいマスタリングでも紫レーンに偏りすぎないよう調整しています。

帯域や重みを変えたい場合は [tools/generate_chart.py](tools/generate_chart.py) の `assign_lane_by_spectrum` 関数を編集 → `python tools/batch_import.py --force` で再生成してください。

### ゲームエンジン（JavaScript / Web Audio API）

- **時刻管理**: `AudioContext.currentTime` ベース（performance.now ではなく音声クロック）
- **判定**: 入力時刻と最も近い未ヒットノーツの誤差で PERFECT/GREAT/GOOD/MISS を決定
- **背景演出**: 単一Canvasに統合描画（ディスコライト・レーザー・パーティクル・ブロック型イコライザー・パースペクティブグリッド）
- **イコライザー**: `AnalyserNode` で取得した周波数データを40バンドに対数集約

### ファイル構成

```
neon-beat/
├── import/                  ← WAV投入フォルダ
├── public/
│   ├── songs/              ← 生成された曲データ
│   └── se/                 ← 効果音（任意）
├── src/
│   ├── App.jsx             ← ステートマシン
│   ├── components/         ← UIコンポーネント
│   ├── engine/             ← AudioEngine, GameEngine など
│   └── utils/              ← 定数・ヘルパー
├── tools/
│   ├── generate_chart.py   ← 単一WAV→譜面
│   └── batch_import.py     ← 一括処理
├── README.md
└── MANUAL.md (このファイル)
```
